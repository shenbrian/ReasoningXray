# Phase 2 Documentation Governance

Phase 2 is the **Language Layer** of ReasoningXray.

Its purpose is to preserve clinical epistemic intent during language rendering.

## Folder meanings

### Phase2_Runtime
These files describe the current runtime contract and implementation-facing architecture.

They are the highest-authority Phase 2 documents after the code itself.

### Phase2_Strategy
These files describe rollout thinking, pilot planning, and execution approach.

They are not runtime law.

### Phase2_Calibration
These files support evaluation, tuning, and sentence-quality assessment.

They do not define architecture.

### Phase2_Foundation
These files explain why the Language Layer exists and what problem it solves.

They guide philosophy and scope, but do not override runtime behavior.

## Governance rule

If a Phase 2 document conflicts with the current codebase:

- the code is the source of truth
- the document must be updated or marked outdated

## Scope protection

Phase 2 is not:

- a new reasoning engine
- a counseling layer
- a behavioral adaptation system
- a predictive medical assistant

It is a translation fidelity and language guardrail subsystem.

## Archive rule

Archived Level 6 temporal-signal files are preserved as research only.

They must not be reintroduced into runtime design without a new charter.
